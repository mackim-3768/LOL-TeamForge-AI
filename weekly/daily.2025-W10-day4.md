# Daily Product Idea Log (2025-W10-day4)

## 1. Discovery (Ideas)

### 1. 소환사 플레이 스타일 레이더 차트 (Frontend Visualization)
- **위치:** 소환사 상세 페이지 (Summoner Detail)
- **문제:** 현재 통계 데이터는 숫자로만 나열되어 있어, 사용자가 자신의 강점과 약점을 한눈에 파악하기 어렵습니다. "내가 공격적인가? 운영 중심인가?"를 직관적으로 알 수 없습니다.
- **가치:** 시각적 정보 전달(Visualization)을 통해 사용자 경험(UX)을 크게 개선하며, 포트폴리오 관점에서도 시각적 완성도를 높여주는 중요한 요소입니다. Insight depth를 강화합니다.

### 2. 최근 20게임 승률/KDA 추세 그래프 (Dashboard)
- **위치:** 소환사 대시보드
- **문제:** 현재 성적이 상승세인지 하락세인지 파악하기 어렵습니다. 단순 평균값은 최근의 변화를 반영하지 못합니다.
- **가치:** 시간의 흐름에 따른 성과 변화를 보여주어 사용자에게 동기를 부여하고, 분석의 깊이를 더해줍니다.

### 3. AI 분석 근거 상세 보기 (Explainability)
- **위치:** AI 분석 결과 모달/패널
- **문제:** AI가 "이 챔피언을 추천합니다"라고 했을 때, 구체적으로 어떤 데이터(예: 라인전 지표, 한타 기여도)에 기반했는지 알 수 없어 신뢰도가 떨어질 수 있습니다.
- **가치:** AI의 판단 근거를 명시적으로 보여줌으로써 "설명 가능한 AI(XAI)" 요소를 강화하고 신뢰도를 높입니다.

### 4. 챔피언 숙련도 기반 맞춤형 팀 추천 (Personalization)
- **위치:** 팀 빌더 (Team Builder)
- **문제:** 단순히 조합상 좋은 챔피언을 추천받아도, 사용자가 해당 챔피언을 전혀 다룰 줄 모르면 실질적인 도움이 되지 않습니다.
- **가치:** 사용자의 숙련도 데이터를 반영하여 실질적으로 사용 가능한 추천을 제공함으로써 효용성을 높입니다.

### 5. API 요청 상태 및 할당량 대시보드 (Admin/DX)
- **위치:** 관리자(Admin) 페이지
- **문제:** Riot API Key 만료나 요청 한도 초과 여부를 로그를 뒤져보지 않으면 알기 어렵습니다.
- **가치:** 운영 효율성과 개발자 경험(DX)을 개선하여 안정적인 서비스 운영을 돕습니다.

## 3. Spec Draft (Top 1)

### Feature Name
**Summoner Playstyle Radar Chart (소환사 플레이 스타일 레이더 차트)**

### Problem Statement
사용자는 자신의 플레이 스타일(전투 지향, 운영 지향, 시야 장악 등)을 직관적으로 파악하고 싶어하지만, 현재는 텍스트 기반의 수치만 제공되어 해석이 어렵습니다.

### MVP Scope
1.  **5가지 핵심 지표 정의:**
    -   **전투 (Combat):** KDA, 분당 딜량(DPM) 기반
    -   **성장 (Farming):** 분당 CS (CSPM), 골드 획득량
    -   **시야 (Vision):** 시야 점수, 제어 와드 구매
    -   **생존 (Survival):** 데스 수 (역산), 생존력
    -   **오브젝트 (Objective):** 타워/드래곤 관여율 (데이터 확보 가능한 경우) 또는 어시스트
2.  **데이터 정규화:** 각 지표를 0~100 스케일로 변환하는 로직 (백엔드 또는 프론트엔드).
3.  **시각화:** `Recharts` 또는 유사 라이브러리를 사용하여 소환사 상세 페이지 상단에 육각형/오각형 레이더 차트 표시.

### Optional Scope
-   티어 평균 대비 내 위치 표시 (비교 분석).
-   차트 영역 호버 시 세부 수치 툴팁 제공.

### Acceptance Criteria
-   [ ] 소환사 상세 페이지 진입 시 레이더 차트가 렌더링되어야 한다.
-   [ ] 5가지 축(전투, 성장, 시야, 생존, 오브젝트/팀워크)에 대한 데이터가 올바르게 매핑되어야 한다.
-   [ ] 데이터가 없는 경우(초기 상태) 빈 차트나 안내 메시지가 표시되어야 한다.
-   [ ] 모바일/데스크탑 환경에서 차트가 깨지지 않고 반응형으로 표시되어야 한다.
