# 주간 기능 계획 - 2025-W10

## 1. 탐색 (아이디어)

### 1. 시각적 역할군 성과 (레이더 차트)
- **적용 대상**: 소환사 상세 페이지 (프론트엔드).
- **문제점**: 사용자는 5개 역할군에 대한 숫자로 된 표만 보게 되어, 빠르게 파악하고 비교하기 어렵습니다.
- **중요한 이유**: 레이더 차트는 플레이어의 프로필(예: "올라운더" vs "원챔 장인")을 즉각적으로 시각화해 줍니다. 포트폴리오의 시각적 매력을 크게 향상시킵니다.
- 적용 여부: Status: DONE

### 2. 전적 히스토리 투명성 목록
- **적용 대상**: 소환사 상세 페이지 (프론트엔드/백엔드).
- **문제점**: 사용자는 집계된 점수만 볼 뿐 원본 매치 데이터를 확인할 수 없어, 데이터에 대한 신뢰가 부족할 수 있습니다.
- **중요한 이유**: 투명성을 높이고 사용자가 점수가 낮은 이유를 디버깅할 수 있게 합니다(예: "아, 서폿 티모한 판이 포함됐네").
- 적용 여부: Status: DONE

### 3. 듀오 시너지 계산기
- **적용 대상**: 새로운 "도구" 섹션 또는 팀 빌더.
- **문제점**: 플레이어들은 종종 듀오로 게임을 하지만, 플레이 스타일이 충돌하는지 모릅니다(예: 바텀 라인의 두 수동적인 플레이어).
- **중요한 이유**: 분석 범위를 개인 성과를 넘어 팀 역학으로 확장하며, 이는 주요 차별화 요소가 됩니다.
- 적용 여부: Status: TODO

### 4. "플레이 스타일 태그" 시스템
- **적용 대상**: 소환사 프로필 헤더.
- **문제점**: 숫자 점수(승률, KDA)는 누군가가 *어떻게* 플레이하는지 설명하지 못합니다(예: "성장형", "공격적", "시야 장악").
- **중요한 이유**: 분석을 더 "인간적"이고 통찰력 있게 만드는 정성적 설명자를 추가합니다.
- 적용 여부: Status: TODO

### 5. AI 분석가 페르소나 선택기
- **적용 대상**: 분석 요청.
- **문제점**: 기본 AI 분석이 일반적이거나 반복적으로 느껴질 수 있습니다.
- **중요한 이유**: 사용자가 페르소나(예: "독설가", "프로 코치", "격려하는")를 선택할 수 있게 하면 기능이 더 재미있고 매력적이 되며, 프롬프트 엔지니어링 기술을 보여줄 수 있습니다.
- 적용 여부: Status: TODO

## 2. 선별 (Triage)

- **선정된 기능**: **시각적 역할군 성과 (레이더 차트)**
- **이유**:
  - **포트폴리오 가치**: 높음. 레이더 차트는 시각적으로 강렬하며 훌륭한 스크린샷이 됩니다.
  - **실현 가능성**: 높음. 백엔드는 이미 모든 역할군에 대해 정규화된 0-100 점수를 제공합니다. 이는 순수하게 프론트엔드 시각화 작업입니다.
  - **사용자 가치**: 플레이어의 다재다능함을 즉시 명확하게 파악할 수 있습니다.

- **선택되지 않은 다른 아이디어들**:
  - *전적 히스토리*: 필수적이지만 차트에 비해 시각적으로 "지루함". 다음 주기에 좋은 후보입니다.
  - *듀오 시너지*: 새로운 백엔드 로직과 더 복잡한 쿼리가 필요할 수 있습니다.
  - *태그*: 무작위성을 피하기 위해 잘 정의된 로직 세트가 필요합니다.
  - *AI 페르소나*: 있으면 좋지만, 시각적 차트가 핵심적인 "한눈에 보기" 분석을 더 잘 개선합니다.

## 3. 사양 초안: 시각적 역할군 성과 (레이더 차트)

### 기능명
시각적 역할군 성과 (레이더 차트)

### 문제 정의
현재 "소환사 상세" 페이지는 성과 점수를 목록/표 형식으로 표시합니다. 이로 인해 사용자가 플레이어의 역할군 다재다능함이나 주력 분야(예: "이 플레이어는 정글 주포인가 아니면 올라운더인가?")를 즉시 이해하기 어렵습니다.

### MVP 범위
- 차트 라이브러리 설치 (예: `recharts` 또는 `react-chartjs-2`).
- `RoleRadarChart` 컴포넌트 생성.
- 이 컴포넌트를 `SummonerDetail` 페이지의 점수 목록 위에 통합.
- 기존 API 응답(`/summoners/{name}/scores`)을 차트 데이터 형식으로 매핑.
- 축: TOP, JUNGLE, MIDDLE, BOTTOM, UTILITY.
- 데이터 포인트: `score` (0-100).

### 선택적 범위 (있으면 좋은 것)
- 비교를 위한 "클래스 평균" 모양 오버레이 (백엔드 통계 필요).
- 마우스 오버 시 정확한 점수를 보여주는 툴팁.

### 인수 조건
1. 소환사 상세 페이지에 레이더 차트가 표시된다.
2. 차트에는 5개 LoL 역할군에 해당하는 5개의 축이 있다.
3. 모양은 API의 `score` 값을 정확하게 반영한다.
4. 역할군 데이터가 없는 경우(점수 0/누락), 0으로 표시된다.
5. 차트는 반응형이며 UI 레이아웃 내에 잘 맞는다.
